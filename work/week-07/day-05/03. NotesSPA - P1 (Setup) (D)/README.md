## NOTES-APP (spa)

Each part of the Notes-App lab will follow the same information covered in class each day. It gives students an opportunity to go through the same steps on their own, but also to make personal decisions about your own website and how it should look. Feel free to make it your own!

## Part 1 - Project Setup
This will directly follow the guide from CatCollector to setup your basic project structure and configuration items.

## Part 2a - Notes Model & CRUD
This will directly follow the guide from Cat Collector to implement the Notes model (like a cat), but you will create the fields (properties) you think a note should have.

## Part 3a - One:Many
You will choose ONE model to implement... Decide which one you would like to do and figure out which properties you think would be useful to include. 

 - Checklist
   - allows a user to add tasks / checklist to a note

 - Reminders 
   - set reminders based on the note

 - Comments
   - users can leave comments on a note

 - Revisions 
   - track file changes / previous versions

## Part 3b - Many:Many
 - Locations
   - Users can create new location for a note to refer to

 - Categories/Tags
   - create categories/tags that are searchable

 - CalendarEntry (Event) 
   - create a calendar entry / event that notes can be related to for that date

 - EmojiReact / Reactions 
   - create a reaction to leave on notes


## Part 4 - One:One
Choose one of the following one-to-one relationships to implement:

- Settings
  - public, private, colors, fonts, etc

- Photos / Gifs
  - save the url of the photo as a string

- Pinned Notes (Pin)
  - user can pin specific notes to be seen in main screen

- NoteStats / ActivityLog
  - reads, writes, updates, views, pins, 

## Part 5 - Authentication
This will directly follow the guide from Cat Collector to implement the user model and authentication system.
